# 🎯 1984 | Global & Competitive Bot

Telegram-бот соревновательной экосистемы «1984 Bounties Competitive» для игроков osu!.

> *"Вы больше не участник — вы стали частью самой системы."*

---

## 📖 О проекте

**1984 | Global & Competitive** — специализированная экосистема для игроков osu!, объединяющая аналитику прогресса, систему еженедельных баунти и рейтинговые дуэли на картах.

Система создана для:
- 🎮 Стимулирования игрового прогресса и выхода из зоны комфорта;
- 🏆 Формирования здоровой соревновательной и развлекательной экосистемы;
- 📊 Отслеживания прогресса через Hunter Progression System (HPS);
- ⚔️ Рейтинговых дуэлей (DUEL) на едином TrueSkill-рейтинге;
- 🎁 Вознаграждения за мастерство, выдержку и лояльность.

## ⚔️ Система дуэлей (DUEL)

Рейтинговая система 1v1 дуэлей на авто-собираемом пуле карт. У каждого игрока **единый TrueSkill-рейтинг** на режим (`casual` / `ranked`): нормальное распределение навыка с матожиданием **μ** и неопределённостью **σ**. После каждой дуэли обе величины обновляются библиотекой `trueskill` из бинарного исхода (победа/поражение); σ постепенно сжимается по мере накопления игр.

Для лидерборда и дивизионов используется **консервативная оценка** `conservative = μ − 3σ` — то, что система уверена, что игрок «как минимум» умеет. TrueSkill выбран намеренно (а не Elo), чтобы в будущем расширить дуэли на команды, ничьи и partial-play (учёт маржи по раундам).

### Формат дуэли

1. `duel <ник> [casual|ranked]` → вызов; соперник принимает или отклоняет (есть мягкое предупреждение при большой разнице дивизионов).
2. **Accept**: по среднему рейтингу обоих определяется целевая сложность, и пул **авто-собирается под уровень игроков** — **5 карт** в Casual, **10** в Ranked. Без pick/ban-фазы и без адаптивной сложности.
3. Создаётся **IRC-комната** (обязательно), оба получают инвайт; бот сам ведёт матч и читает результаты раундов из osu! API.
4. **Раунд-луп** по картам пула: Casual — **Bo5** (до 3 побед), Ranked — **Bo10** (до 6). Если пул исчерпан без достижения цели — победа по числу выигранных раундов; при равенстве играется тай-брейк-карта.
5. **Финиш**: `update_ratings` (TrueSkill), смена дивизиона по conservative, карточка результата, IRC-комната закрывается.
6. Активные дуэли **переживают перезапуск бота** — состояние восстанавливается из БД, включая собранный пул и матч.

### Подсчёт раунда (хардкор)

Очко за раунд начисляется **только если игрок прошёл карту**:

- Прошли оба → очко тому, у кого **больше score**.
- Прошёл один → очко ему (соперник зафейлил — очко не получает).
- Оба зафейлили → **пустой раунд**, очко никому.

### Дивизионы

Ranked-рейтинг раскладывается на **5 тиров по 3 дивизиона** (15 ступеней), от **Cadence III** (низ) до **Rhythmus I** (топ), по порогам `conservative`. Смена дивизиона по итогам дуэли публикуется карточкой (в т.ч. в отдельный чат, если задан через `setduelnotifychat`).

### Глубокий анализ карт (Deep Analysis)

Классификатор карт сохранён — он питает и авто-сборку пула, и HPS-формулу (Φ). При добавлении карты в пул (`dueladdmap`) бот:

1. **Скачивает `.osu` файл** и парсит паттерны: стримы/бёрсты/death streams, углы и back-and-forth движения курсора, скорость прыжков, дисперсию SV и плотности нот, HP drain.
2. **Запрашивает osu! API** (`/beatmaps/{id}/attributes`): `aim_difficulty`, `speed_difficulty`, `slider_factor`.
3. **Тегирует карту** по типу (`aim` / `speed` / `acc` / `cons` / `mixed`) и считает per-axis skill-stars — это используется при подборе сбалансированного пула под уровень игроков.

### Калибровка рейтинга (μ-сид по PP)

Стартовый μ берётся из osu! pp по кусочно-линейной кривой (σ при этом всегда стартует с полной неопределённости, так что conservative «разгоняется» только по мере игр). Первые **10 матчей** — фаза размещения (placement). Шкала μ примерно совпадает со шкалой порогов дивизионов (0…~4300):

| PP | ≈ стартовый μ |
|----|---------------|
| 0 | 900 |
| 3 000 | 1 450 |
| 5 000 | 1 750 |
| 10 000 | 2 400 |
| 18 000 | 3 000 |
| 35 000 | 3 800 (кап) |

## 🏆 Система рангов (HPS)

| Ранг | Диапазон HP | Описание |
|------|-------------|----------|
| 🔵 **Candidate** | 0–249 | Испытуемый. Способности замечены, лояльность не доказана. |
| 🟢 **Member** | 250–749 | Полноправная единица структуры. Стабильно функционируете в баунти-циклах. |
| 🟠 **Inspector** | 750–1499 | Опытный субъект. Право контроля над ритмом и конкуренция с элитой. |
| 🔴 **Commissioner** | 1500–2999 | Высшее руководство. Авторитет незыблем, рекорды эталонные. |
| 🟣 **Big Brother** | 3000+ | Абсолютный предел. Вы больше не участник — вы стали частью самой системы. |

Каждый ранг разделён на **4 дивизиона** (IV → I). Итого 20 дивизионов. Big Brother I начинается с **5 000 HP** (последний шаг BB II→I — 1 000 HP).

## 🎮 Баунти-система

### Еженедельный пул

Каждую неделю бот автоматически генерирует 24 баунти — по 6 на каждый тир:

| Тир | Диапазон ★ | Доступен |
|-----|-----------|----------|
| **C** | 2.0–4.5★ | Игрокам своего тира |
| **B** | 4.5–7.0★ | Игрокам своего тира |
| **A** | 7.0–10.0★ | Игрокам своего тира |
| **Open** | Полный диапазон | Всем |

Тир игрока определяется по HPS-очкам: C (0–749 HP), B (750–1499 HP), A (1500+ HP).

**Лимиты:** 6 авто-баунти в неделю на игрока, 3 попытки на каждый баунти.

### Типы баунти

| Тип | Условие | Множитель T |
|-----|---------|------------|
| **SS** | 100% точность | ×1.6 |
| **Metronome** | Наименьший Unstable Rate | ×1.4 |
| **Accuracy** | Высокая точность | ×1.2 |
| **Marathon** | Длинная карта | ×1.2 |
| **Mod** | Прохождение с модами | ×1.1 |
| **Pass** | Пройти карту | ×1.0 |
| **First FC** | Первый Full Combo | ×1.0 |

## 🧮 Формула начисления HP (v2)

```
HP_final = Base · Φ(DUEL_map) · Ψ(Δ) · Λ(t) · C_pen · R · T  +  Vanguard
```

Результат сжимается soft-cap через `tanh`: `600 · tanh(HP_pre / 600)` — очень сильные скоры приближаются к 600, но никогда не достигают его жёстко.

| Компонент | Формула | Смысл |
|-----------|---------|-------|
| **Base** | 60 | Базовое значение |
| **Φ(DUEL)** | `0.5 + 0.05 · DUEL^1.8` | Сложность карты |
| **Ψ(Δ)** | `0.5 + 1.5 / (1 + exp(−1.5·Δ))` | Скилл-гэп (Δ = DUEL_карты − DUEL_игрока) |
| **Λ(t)** | `max(0.4, ln(1 + t/150) + 0.6)` | Масштаб по длине карты |
| **C_pen** | `√(combo/max_combo) · exp(−3·miss_rate)` | Штраф за промахи и дроп комбо |
| **R** | см. таблицу | Тип результата |
| **T** | см. таблицу типов | Тип баунти |
| **Vanguard** | +25 HP | Бонус первому игроку |

### Результат (R)

| Тип | Множитель |
|-----|-----------|
| Win | ×1.5 |
| Condition | ×1.0 |
| Partial | ×0.5 |
| Participation | ×0.2 |

> **Anti-farm:** повторное прохождение одной и той же карты снижает множитель. **Bootstrap:** в первые 90 дней активности игрок получает до ×1.5 к начислениям.
